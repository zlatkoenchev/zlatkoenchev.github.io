Introduction

-       UPS for home light with unique search mode and low power – started 2020

BMS

I start to work on this project because of package of e.g. 80 LiIon 2170 cells which got by occasion. Unfortunately I cannot find good BMS (good means to be able to set min/max battery voltage (I want to charge to e.g.4..4.1V instead 4.2V) low consumption, balancing to work all the time)

I decide to use AVR AtTiny13 MCU for cell node. Currently only FW is ready (but not tested on real HW). I plan to use UART on 9600bps with autobaudrate adjustment to compensate AVR clock jitter.

There should be one main cell node (probably PIC based) which to control charge, discharge and to communicate with other cell nodes.

HOME Automation system

I decide to to create my own system after analyze of commercial ones (Z-wave, Zigbee, Wifi, etc). Most of them needs hub or Internet connection to work and consume so much power, that you have to stop you water heater for 1..2 days every month to compensate the self consumption!

My system is standalone, peer to peer and consumes just 0.1W (1.5VA) per node powered from 220V. I have also kinetic powered nodes, CR2045 battery powered nodes and last one is TEG thermo valve (powered by thermal generator)

 

UPS for home lights

The idea for this project popups from a storm – the electricity of the house of my wife stops and we start to search for devices like phones and tables which have backlight in order to move in the dark…We have super modern phones ot tablets, but we stay in the dark? I decide to solve the problem. First I think that switching to 12V DC power for lights is the best option because, because the battery is just connected the lines and that’s all. But after some thinking and calculations, I decide to use 220V UPS.
